# Mission: 六边形架构（Hexagonal Architecture / Ports & Adapters）

## Why
把端口/适配器方法论从「直觉」升级为「能讲清、能评审、能动手改」的架构能力（架构师方向补课）；
并用它逐步改造一个现有 C++ 项目——不是绿地搭骨架，而是把遗留代码掰成六边形。
本课程是 `~/learning/learning-ddd` DDD 课程的续篇，直接建在已验证的端口/适配器直觉之上。

## Success looks like
- 能画出并讲清六边形模型：驱动侧/被驱动侧、端口即对话、测试与 UI 同为适配器
- 能从六边形视角评审设计：指出业务逻辑泄漏点，并说清与 Clean / Onion / 传统分层的取舍
- 能在现有 C++ 项目上执行一次真实迁移：识别接缝 → 补特征测试 → 抽出第一个端口

## Constraints
- 学习者已完成 DDD 战略+战术课程（仓储端口/适配器、防腐层、分层依赖已内化）——不从零讲
- 代码一律现代 C++（C++17/20），中文授课、代码与注释英文
- 沿用 DDD 课的节奏（快、少铺垫、多判断题）与组件库

## Out of scope
- 微服务拆分、CQRS/Event Sourcing（与 DDD 课一致，基础扎实后再评估）
- 具体框架选型（HTTP 库、DI 框架等）——先掌握手工组装
